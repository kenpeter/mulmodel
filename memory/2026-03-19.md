### Key Issues
1. Generated code has truncated keywords: `vec` instead of `vector`, `ve` instead of `vector`
2. Repetition within functions
3. Undefined variables (using wrong variable names)
4. Incomplete statements (truncated mid-expression)
5. Wrong algorithm for the problem (generates code for wrong problem)
6. `<|endoftext|>` token bleeding into generated code

### Architecture
- BigModel: 253M params, emb_dim=1024, n_layers=16, n_heads=16, n_blocks=4
- tiktoken gpt2 vocab (50257)
- ctx_len=384 (barely fits, OOM at ctx_len=512)
- Training: lr=5e-4, batch_size=4, grad_accum=8, weight_decay=0.01

### Postprocessing (eval_cf_real.py)
- Fixed regex errors in postprocess_code (removed fragile regex patterns)
- Simple string replacements for common patterns (++++, >>>, >>>>)
- Still not enough to fix fundamentally broken generated code

### Thoughts
- Original CoT model got 80% pass_rate at loss 2.03 after many thousands of steps
- [CODE] format model at loss 1.72 is 0% — format is harder to learn
- Need MUCH more training on [CODE] format
- Maybe [CODE] format is fundamentally inferior to CoT format
- Consider going back to CoT format or using a hybrid approach
