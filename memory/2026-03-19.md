## Key Insight: The Original 80% Result
The original byte-level CoT model got 80% pass_rate at loss 2.03. How?
- Byte-level output: `bytes(ids[0].tolist()).decode("utf-8", errors="replace")`
- This SILENTLY replaced invalid UTF-8 bytes with `U+FFFD` (`\ufffd`)
- Code like `<bits/stdc++++.h>` came from invalid byte combinations
- The `postprocess_code()` function fixed common patterns: `++++` → `++`
- The original eval didn't strip/replace invalid bytes properly — it kept them as-is
- So the "broken" byte-level code actually had valid UTF-8 that g++ could compile!

## Critical Bug in Current Eval
eval_cf_real.py uses `tokenizer.decode()` which replaces invalid bytes with replacement characters. This HIDES the corruption. The original eval used `bytes(...).decode("utf-8", errors="replace")` which is the correct approach for byte-level models.
